Almacenamiento de valores con el operador de asignación

En JavaScript, puedes almacenar un valor en una variable con el operador de asignación (=).

            myVariable = 5;

Esto asigna el Number valor 5 a myVariable.

Si hay cálculos a la derecha del operador =, estos se realizan antes de que se asigne el valor a la variable a la izquierda del operador.

            var myVar;
            myVar = 5;

En primer lugar, este código crea una variable llamada myVar. Luego, el código asigna 5 a myVar. Ahora, si myVar aparece nuevamente en el código, el programa lo tratará como si fuera 5.

Asignar el valor 7 a la variable a.

            var a;
            a = 7;

Asignar el valor de una variable a otra

Después de asignar un valor a una variable mediante el operador de asignación (=), puede asignar el valor de esa variable a otra variable mediante el operador de asignación.

            var myVar;
            myVar = 5;
            var myNum;
            myNum = myVar;

Lo anterior declara una myVar variable sin valor y luego le asigna el valor 5. A continuación, se declara una variable denominada myNum sin valor. Luego, el contenido de myVar (que es 5) se asigna a la variable myNum. Ahora, myNum también tiene el valor de 5.

Asignar el contenido de a a la variable b.

            var a;
            a = 7;
            var b;
            b = a;

Inicialización de variables con el operador de asignación

Es común inicializar una variable a un valor inicial en la misma línea en que se declara.

            var myVar = 0;

Crea una nueva variable llamada myVar y le asigna un valor inicial de 0.

Defina una variable a con var e inicialícela con un valor de 9.

            var a = 9;

Declarar variables de cadena

Anteriormente utilizaste el siguiente código para declarar una variable:

            var myName;

Pero también puedes declarar una variable de cadena de esta manera:

            var myName = "your name";

"your name" se denomina literal de cadena. Un literal de cadena, o cadena, es una serie de cero o más caracteres encerrados entre comillas simples o dobles.

Cree dos nuevas variables de cadena: myFirstName y myLastName y asígneles los valores de su nombre y apellido, respectivamente.

            var myFirstName = "TuNombre";
            var myLastName = "TuApellido";

Comprensión de las variables no inicializadas

Cuando se declaran variables de JavaScript, tienen un valor inicial de undefined. Si realiza una operación matemática en una variable undefined, el resultado será NaN, lo que significa "No es un número". Si concatena una cadena con una variable undefined, obtendrá una cadena de undefined.

Inicialice las tres variables a, b, y c con 5, 10, y "I am a" respectivamente para que no sean undefined.

            var a = 5;
            var b = 10;
            var c = "I am a";

Explorar las diferencias entre las palabras clave var y let

Uno de los mayores problemas al declarar variables con la var palabra clave es que puedes sobrescribir fácilmente las declaraciones de variables:

            var camper = "James";
            var camper = "David";
            console.log(camper);

En el código anterior, la camper variable se declara originalmente como James y luego se reemplaza por David. Luego, la consola muestra la cadena David.

En una aplicación pequeña, es posible que no te encuentres con este tipo de problema. Pero a medida que tu base de código se hace más grande, es posible que sobrescribas accidentalmente una variable que no era tu intención. Debido a que este comportamiento no genera un error, la búsqueda y reparación de errores se vuelve más difícil.

En ES6, una actualización importante de JavaScript, se introdujo una palabra clave llamada let para resolver este problema potencial con la var palabra clave.

Si reemplaza var con let en el código anterior, se producirá un error:

            let camper = "James";
            let camper = "David";

El error se puede ver en la consola de su navegador. Entonces, a diferencia de var, cuando se usa let, una variable con el mismo nombre solo se puede declarar una vez.

Actualice el código para que solo use la let palabra clave:

            let camper = "James";
            camper = "David";
            console.log(camper);

Comprender la distinción entre mayúsculas y minúsculas en las variables

En JavaScript, todos los nombres de funciones y variables distinguen entre mayúsculas y minúsculas, lo que significa que el uso de mayúsculas y minúsculas es importante.

MYVAR no es lo mismo que MyVar ni myvar. Es posible tener múltiples variables distintas con el mismo nombre pero con diferente uso de mayúsculas y minúsculas. Se recomienda encarecidamente que, por razones de claridad, no utilice esta función del lenguaje.

Mejores prácticas

Escriba los nombres de las variables en JavaScript en camelCase. En camelCase, los nombres de variables de varias palabras tienen la primera palabra en minúscula y la primera letra de cada palabra subsiguiente en mayúscula.

Ejemplos:

            var someVariable;
            var anotherVariableName;
            var thisVariableNameIsSoLong;
